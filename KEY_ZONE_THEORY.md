# 历史多次触及关键区间识别 — 理论详解

## 一、为什么关键区间有效

### 1.1 市场记忆效应
价格在某个位置反复停留，说明市场参与者在集体"记住"这个位置。本质是**订单簿的聚类效应**：

```
时间线：
─────────┬─────────┬─────────┬─────────
  触及1    触及2    触及3    触及4（突破/反弹）

每次触及 → 部分止损/止盈触发 → 部分新单入场
反复触及 → 该价位积累了高密度订单 → 形成"记忆"
```

### 1.2 三类参与者的行为叠加

| 参与者 | 行为 | 对区间的贡献 |
|--------|------|-------------|
| **趋势交易者** | 突破时加仓/止损 | 触及越多，止损越密 |
| **反转交易者** | 触及后反向开仓 | 触及越多，挂单越厚 |
| **做市商** | 区间内提供流动性 | 维持区间稳定 |

触及次数越多 → 三种力量叠加 → 区间越"硬"。

## 二、区间的四个维度

一个关键区间的强度由四个维度决定，不是只有"触及次数"：

### 维度1：触及次数（Touch Count）
最直观的维度，也是我们代码中优先使用的。

| 触及次数 | 强度评级 |
|---------|---------|
| ≥5次 | ⭐⭐⭐ 核心关键位 |
| 3-4次 | ⭐⭐ 重要参考位 |
| 2次 | ⭐ 初步形成 |
| 1次 | 仅摆动点，不算区间 |

### 维度2：触及质量（Touch Quality）
不是所有触及都一样。看每次触及的反应强度：

```
高质量触及：价格快速反弹/回落，K线实体大，影线短
低质量触及：价格磨蹭穿越，影线长，实体小
```

衡量方法：
- 触及后 3 根 K 线的最大反向幅度 / ATR
- >2x ATR 的反向移动 = 高质量触及
- <1x ATR 的反向移动 = 低质量触及

### 维度3：时间跨度（Time Span）
区间跨越的时间越长，越可靠。

| 时间跨度 | 强度 |
|---------|------|
| >30天 | 月线级关键位，极少失效 |
| 7-30天 | 周线级，可靠 |
| 1-7天 | 日线级，有效但可被突破 |
| <1天 | 日内级，参考性弱 |

### 维度4：价格行为的戏剧性（Price Drama）
在区间位置是否发生过显著的"戏剧性"事件：

- **大阳/大阴线**（>3x ATR）：说明此处有强烈方向性意图
- **长影线反转**（影线 > 2x 实体）：说明此处有强对抗
- **密集成交区**：价格在此处长时间横盘（多根K线收盘接近）

## 三、区间聚类算法 — 三遍扫描法

当前代码用的是简单距离聚类。更精确的做法是三遍扫描：

### 第一遍：摆动点检测
```
条件：当前K线的高点 > 左右各2根的高点 → 摆动高点
      当前K线的低点 < 左右各2根的低点 → 摆动低点
```
检测所有摆动点，标记时间戳。

### 第二遍：密度聚类（DBSCAN思想）
```
聚类半径 = ATR × 1.0（同向摆动点）
最小样本 = 2（至少2个点才成区）

对于每对相邻摆动点：
  if |p1 - p2| < cluster_radius:
    合并为一个区间的候选
```

### 第三遍：区间评估与排序
对每个候选区间计算：
```
strength_score = 
    触及次数归一化 × 0.4 +
    触及质量(平均反向幅度/ATR) × 0.25 +
    时间跨度归一化 × 0.2 +
    价格戏剧性(是否有大K线/长影线) × 0.15
```

## 四、区间与SuperTrend的关系

这是我们策略的核心创新点——**双确认机制**：

### 场景A：区间远≠ST → 低置信度
```
区间 [上方]     ← 历史阻力
  |
  |  <-- 距离大，可能只是历史巧合
  |
ST [当前]       ← SuperTrend线
价格 [当前]
```
→ 历史阻力和当前趋势线不重叠，置信度低

### 场景B：区间≈ST → 高置信度 ✅
```
区间+ST [同一位置]  ← 历史阻力 + 当前趋势线重合
  |  双重压制 → 极强阻力
  |
价格 [当前]
```
→ 历史记忆和当前趋势在同一位置形成合力，置信度高

### 实现方式
在 `find_key_zones()` 中，我们已经做到了：
```python
# 优先选区间中心和SuperTrend接近的
strong.sort(key=lambda z: abs(z["center"] - st_line))
```

## 五、针对不同币种的参数自适应

微盘币（PUMP、HMSTR、GALA 等）价格极低（< $0.01），ATR 可能是价格的 5-10%。这时聚类半径需要调整：

| 币种类型 | price < 0.01 | price 0.01-1 | price > 1 |
|---------|-------------|-------------|-----------|
| 聚类半径 | ATR × 1.5 | ATR × 1.0 | ATR × 0.8 |
| 最小触及 | 2次 | 2次 | 3次 |

因为低价币波动率大，区间本身更宽，需要更大的聚类半径。

## 六、实战验证方法

### 回测验证
对历史数据，在每次区间触及位置模拟：
1. 触及后 N 根 K 线内是否反转（>2x ATR）
2. 统计触及→反转的概率
3. 对比触及次数与反转概率的关系

### 预期关系
```
触及次数  →  反转概率
   2              ~35%
   3              ~50%
   4              ~60%
   5+             ~70%
```

如果实际数据显示触及3次的反转概率不到40%，说明这个币种的"市场记忆"不可靠，应该降低其权重或跳过。

## 七、代码中可以改进的点

基于以上理论，`strategy_engine.py` 可以增加：

1. **触及质量评分** — 记录每次触及后的反转幅度
2. **时间跨度权重** — 老区间比新区间更重
3. **ST-区间距离加权** — 区间中心离ST越近，置信度越高
4. **微盘币自适应半径** — 根据价格量级调整聚类范围
5. **"戏剧性"加分** — 检测区间位置是否有大阳大阴线
